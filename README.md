# Español CRM — публікація на GitHub Pages

У мене немає підключеного GitHub-акаунту в цій сесії, тому запушити напряму я не можу. Але ось `index.html`, готовий для завантаження — виклади його вручну за 2 хвилини:

## Кроки

1. Зайдіть на [github.com](https://github.com) → **New repository** (наприклад, `espanol-crm`) → публічний → Create repository.
2. На сторінці репозиторію натисніть **Add file → Upload files**, перетягніть `index.html`, натисніть **Commit changes**.
3. Перейдіть у **Settings → Pages**. У розділі **Build and deployment** оберіть Source: **Deploy from a branch**, Branch: **main**, папка **/(root)** → Save.
4. За 1-2 хвилини застосунок буде доступний за адресою:
   `https://<ваш-логін>.github.io/espanol-crm/`
5. Відкрийте це посилання на телефоні/планшеті в Safari чи Chrome → **Додати на головний екран**.

## Важливо про дані

GitHub Pages хостить лише сам файл — це все ще статичний застосунок без сервера. Дані (`localStorage`) зберігаються окремо в кожному браузері/пристрої, тобто:

- посилання можна відкрити з будь-якого пристрою, але
- дані НЕ синхронізуються між пристроями (телефон і планшет матимуть різні бази).

Якщо потрібен спільний доступ для кількох викладачів з синхронізацією — знадобиться справжній бекенд (Airtable, Firebase, Supabase тощо). Скажіть, і я підготую таку версію.
